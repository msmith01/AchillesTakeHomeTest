# Earnings Call RAG Pipeline

A fully local RAG (Retrieval-Augmented Generation) pipeline over AAR Corp earnings call transcripts (2018–2022). Ingests PDFs, chunks them recursively by tokens, embeds with a local model, stores in a vector database, re-ranks with a cross-encoder, and generates cited answers via a local LLM.

---

## Pipeline Overview

```
PDFs → Chunk (recursive, token-overlap) → Embed (Qwen3-8B) → ChromaDB
                                                                    ↓
Query → Embed → Retrieve top-20 → Re-rank (BGE cross-encoder) → top-5
                                                                    ↓
                                              Generate (qwen2.5:14b via Ollama)
                                                                    ↓
                                         Cited answer + Faithfulness score (LLM-as-judge)
```

---

## Models (all local, no API keys required)

| Role | Model | How |
|------|-------|-----|
| Embeddings | `Qwen/Qwen3-Embedding-8B` | HuggingFace Transformers, bfloat16 on GPU |
| Re-ranker | `BAAI/bge-reranker-v2-m3` | HuggingFace Transformers, bfloat16 on GPU |
| Generation + Judge | `qwen2.5:14b` | [Ollama](https://ollama.com/) |

## Setup

### 1. Install Python dependencies

```bash
pip install -r requirements.txt
```

### 2. Install and start Ollama

Download from [ollama.com](https://ollama.com/), then pull the generation model:

```bash
ollama pull qwen2.5:14b
```

### 3. Build the vector store

Place your earnings call PDFs in `raw_data/`, then run:

```bash
python process_earnings.py
```

This embeds all chunks with Qwen3-Embedding-8B and persists them to `chroma_store/`. Only needs to run once.

> If starting from the raw CSV, run `01_save_earnings.py` first to generate the PDFs.

## Usage

### Streamlit UI

```bash
streamlit run app.py
```

Chat interface with year filter, passage viewer, per-response citation count, faithfulness score, and retrieval score.

### CLI (interactive + evaluation)

```bash
python evaluate.py
```

- Type any question to get a streamed, cited answer with a Rich-formatted passage table.
- Type `eval` to run 10 benchmark questions and write results to `eval.jsonl`.
- Type `quit` to exit.

---

## Evaluation Metrics (`eval.jsonl`)

Each record in `eval.jsonl` contains:

| Field | Description |
|-------|-------------|
| `question` | Input question |
| `answer` | Full LLM response |
| `num_citations` | Count of `[Year: … | Transcript: … | Page: …]` patterns |
| `has_valid_citation` | Boolean |
| `faithfulness_score` | 0.0–1.0 from LLM-as-a-Judge |
| `faithfulness_reason` | One-sentence judge explanation |
| `top_hit_score` | Cross-encoder score of best passage |
| `avg_hit_score` | Mean cross-encoder score across top-k |
| `sources` | List of retrieved chunks with metadata |

**Summary printed after eval run:**
- % of responses with valid citations
- Average citations per response
- Average retrieval score
- Average faithfulness score

## Chunking Strategy

Recursive token-based chunking with a hierarchical fallback:

```
Paragraphs → Sentences → Words → Characters
```

- **Chunk size**: 500 tokens (`cl100k_base` encoding)
- **Overlap**: 50 tokens (~10%) carried forward as a sliding window
- Each chunk retains its source page number via `[PAGE N]` markers injected at ingestion
